Perceptrón¶

Por Camilo A. Aguilar L.¶

El perceptr√≥n es una unidad b√°sica de inferencia en forma de discriminador lineal, a partir de lo cual se desarrolla un algoritmo capaz de generar un criterio para seleccionar un sub-grupo a partir de un grupo de componentes m√°s grande.

La limitaci√≥n de este algoritmo es que si dibujamos en un plot estos elementos, se deben poder separar con un hiperplano √∫nicamente los elementos "deseados" discrimin√°ndolos (separ√°ndolos) de los "no deseados".

En el presente notebook vamos a contruir un perceptr√≥n que nos permita discriminar algunas condiciones de inter√©s, sobre los datos artificiales que fueron recolectados en clase.

El perceptr√≥n usa una matriz para representar las redes neuronales y es un discriminador que traza su entrada x (un vector binario) a un √∫nico valor de salida $f(x)$ (un solo valor binario) a trav√©s de dicha matriz.

\begin{equation*} f(x) = \begin{cases}1 & \text{si }w \cdot x > 0\\0 & \text{en otro caso}\end{cases} \end{equation*}

Donde $w$ es un vector de pesos reales y $w\cdot x$ es el producto escalar (que computa una suma ponderada). El valor de $f(x)$ (0 o 1) se usa para clasificar $x$ como un caso positivo o un caso negativo, en el caso de un problema de clasificaci√≥n binario. La suma ponderada de las entradas debe producir un valor mayor que 0 para cambiar la neurona de estado 0 a 1.

A continuaci√≥n se define la funci√≥n $f$ que evaluar√° las sumas de dicho producto escalar:

In [5]:
f <- function(x){
  if(x>0) return(1)
  return(0)
}

1. Perceptrón simple¶

La intenci√≥n ahora es separar mediante una l√≠nea (plano en 2D), las dos categor√≠as. Para ello definimos el algoritmo de aprendizaje, el cual inicia con unos pesos iniciales definidos como un vector de ceros, y a partir de los cuales eval√∫a la funci√≥n $f$. Los pesos van siendo corregidos usando la regla de aprendizaje del perceptr√≥n simple.

In [7]:
perceptron <- function(x, y, num_pc=-1, iter=10000) {
     require(progress)
     # x: Matriz de datos
     # y: vector de etiquetas sobre x, a discriminar
     # iter: N√∫mero de iteraci√≥nes. Por defecto iter=10000
     
     # num_pc: Siempre que x represente una tranformaci√≥n de los datos originales a partir de componente principales,
     # el par√°metro num_pc podr√° ser usado para seleccionar el n√∫mero de componentes principales a utilizar
     if (num_pc != -1){
          x <- x[, 1:num_pc]
     }
     
     
     # pesos iniciales
     w <- rep(0, dim(x)[2] + 1)
     # Vector de errores
     err <- rep(0, iter)
     a_vec <- rep(NA, iter)
     b_vec <- rep(NA, iter)
     
     # Bucle para completar todas las iteraciones
     pb <- progress_bar$new(total = length(1:iter))
     for (i in 1:iter) {
          # Selecci√≥n aleatoria de individuos
          ind <- sample(1:nrow(X), size=1)
          # Funci√≥n de paso
          z <- sum(w[1:length(w)] * c(as.numeric(x[ind, ]), 1) ) #+ w[1]
          ypred <- f(z)
          # Regla de aprendizaje del perceptr√≥n simple
          w <- w + 1 * (y[ind] - ypred) * c(as.numeric(x[ind, ]), 1)
               
          # Grafica cada w
          a_vec[i] <- -w[3]/w[2] #intercepto
          b_vec[i] <- -w[1]/w[2] #pendiente pasando por el origen
               
          # Funci√≥n que calcula el error de cada iteraci√≥n
          if ((y[ind] - ypred) != 0.0) {
               err[i] <- err[i] + 1
               }
          pb$tick()
     }
     a_vec <- a_vec[!is.na(a_vec)]
     b_vec <- b_vec[!is.na(b_vec)]
     resultados <- list(errores = err, a = a_vec, b = b_vec)
     return(resultados)
}

Se realizan 10000 iteraciones del algoritmo, donde la √∫ltima iteraci√≥n representada por la l√≠nea negra, separa de manera √≥ptima las dos clases.

2. Perceptrón con bolsillo¶

El algoritmo anterior nos permit√≠a hacer uso de una regla de aprendizaje para determinar una l√≠nea (plano) de separaci√≥n. Sin embargo no considera la l√≠nea con el menor error entre todas las iteraciones consideradas, por lo que ahora ralizamos una peque√±a modificaci√≥n al algoritmo, la cual guarda en un objeto determinado (bolsillo), en cada iteraci√≥n la soluci√≥n con menor error de predicci√≥n, que ir√° siendo actualizada √∫nicamente si se encuentra un vector de pesos $w$ con un error inferior al almacenado en el bolsillo. As√≠, el objetivo de esta modificaci√≥n es encontrar un vector de pesos que maximice el n√∫mero de clasificaciones correctas.

## Perceptr√≥n con bolsillo
perceptron_bolsillo <- function(x, y, num_pc=-1, iter=10000) {
     require(progress)
     # x: Matriz de datos
     # y: vector de etiquetas sobre x, a discriminar
     # iter: N√∫mero de iteraci√≥nes. Por defecto iter=10000
     
     # num_pc: Siempre que x represente una tranformaci√≥n de los datos originales a partir de componente principales,
     # el par√°metro num_pc podr√° ser usado para seleccionar el n√∫mero de componentes principales a utilizar
     if (num_pc != -1){
          x <- x[, 1:num_pc]
     }
     
     
     # pesos iniciales
     w <- rep(0, dim(x)[2] + 1)
     # Vector de errores
     err <- rep(0, iter)
     a_vec <- rep(NA, iter)
     b_vec <- rep(NA, iter)
     
     # Bucle para completar todas las iteraciones
     pb <- progress_bar$new(total = length(1:iter))
     for (i in 1:iter) {
          # Selecci√≥n aleatoria de individuos
          ind <- sample(1:nrow(X), size=1)
          # Funci√≥n de paso
          z <- sum(w[1:length(w)] * c(as.numeric(x[ind, ]), 1) ) #+ w[1]
          ypred <- f(z)
          # Regla de aprendizaje del perceptr√≥n simple
          w <- w + 1 * (y[ind] - ypred) * c(as.numeric(x[ind, ]), 1)
          
          # Grafica cada w
          a_vec[i] <- -w[3]/w[2] #intercepto
          b_vec[i] <- -w[1]/w[2] #pendiente pasando por el origen
          
          # Funci√≥n que calcula el error de cada iteraci√≥n
          if ((y[ind] - ypred) != 0.0) {
               err[i] <- err[i] + 1
          }
          pb$tick()
          
          # W bolsillo
          if(i == 1){
               w.bolsillo <- w
          } else{
               if(err[i] < err[i-1]){
                    w.bolsillo <- w
                    
                    # Grafica cada w bolsillo
                    a_vec[i] <- -w[3]/w[2] #intercepto
                    b_vec[i] <- -w[1]/w[2] #pendiente pasando por el origen
                    abline(a, b, col="grey")
                    
               }
          }
     }
     a_vec <- a_vec[!is.na(a_vec)]
     b_vec <- b_vec[!is.na(b_vec)]
     resultados <- list(w.bolsillo = w.bolsillo, errores = err, a = a_vec, b = b_vec)
     return(resultados)
}
